# Character Counter

###### Hi friend, this is my third (self-started) project in Rust on here.

It's a simple command-line character counter written in Rust that calculates the total character count of two user inputs while ignoring whitespaces.

# Description

This program takes two string inputs from the user, trims the whitespaces, and calculates the total number of characters. It demonstrates basic Rust concepts including:

- · · String manipulation and trimming
- · · User input handling with std::io
- · · Basic error handling with .expect()
- · · Conditional logic for input preview
- · · Function definitions and usage

# Features Include

- · · Interactive command-line interface
- · · Whitespace removal from inputs
- · · Input preview with truncation for long strings
- · · Clear, friendly user prompts
- · · Straightforward character counting

# Usage

Run the program using Cargo:

```bash
cargo run
```

The program will prompt you for:

1. Your first text input
2. Your second text input

It will display a preview of your first input (truncated if longer than 10 characters) and then calculate the total character count of both inputs.

# Example Round

```bash
$ cargo run

Ignoring Whitespaces!

from Rust!
Good one bro!

 Thanks for using my Rusty counter, bro!
```

Note: trim() removes only leading (beginning) and trailing (end) whitespace characters, not spaces between words.

# How It Works

1. The program prompts for the first input and reads it from stdin
2. It trims leading/trailing whitespaces from the input
3. If the input is longer than 10 characters, it shows a truncated preview
4. The program prompts for the second input and processes it similarly
5. Both strings are passed to the add() function which returns the sum of their lengths
6. The total character count is displayed

- · add(x: String, y: String) -> usize: Returns the combined length of two strings

- · Preview generation: Conditionally truncates long inputs for display

# Error Handling

The program includes basic error handling for:

- · Failed input reading using .expect("Failed to read input")

# Limitations

- · Only processes two inputs per run
- · Counts all non-whitespace characters equally
- · Basic preview truncation at 10 characters
- · No support for Unicode grapheme clusters (counts bytes/UTF-8 characters)
